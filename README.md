## 5. üîÑIdentify Duplicate Redeems and Calculate Excess Points

### Description

This script analyzes a CSV file to identify duplicate redeems and calculate excess points refunded to customers. It compares the amount of points subtracted during the redeem process against the points added back through refunds. If the refunded points exceed the redeemed points, the script calculates the excess and generates a detailed report.

### Functionality

- Loads a CSV file containing point wallet logs.
- Identifies redeems where the refunded points exceed the points redeemed.
- Calculates the excess points for each redeem where a discrepancy is found.
- Saves the result in a report file, including the `RedeemId`, total refunded points, redeemed points, and the excess points identified.

### Requirements

- CSV file generated by the following SQL query:
  ```sql
  select
      *
  from
      point_wallet_log pwl
  where
      pwl."originCustomerId" = 'originCustomerId'
          and pwl."redeemId" is not null
      or pwl."redeemId" in (
      select
          distinct p."redeemId"
      from
          point_wallet_log p
      where
          p."originCustomerId" = 'originCustomerId'
  );
  ```

## 7. üîÑConvert INSERT Statements to UPDATE Statements

### Description

This script transforms SQL `INSERT` statements into corresponding `UPDATE` statements. It extracts the table name, columns, and values from the `INSERT` statements, then generates `UPDATE` statements by setting the column values and using the `id` as the condition in the `WHERE` clause. The resulting `UPDATE` statements are saved in a `.sql` file.

### Functionality

- Extracts table name, columns, and values from the provided `INSERT` statements.
- Converts the `INSERT` statements into `UPDATE` statements, using the `id` column to update the correct record.
- Saves the generated `UPDATE` statements to a SQL file (`transform-update.sql`).

### Requirements

- Input: A string containing one or more `INSERT` statements to be converted.

### Example

If the `INSERT` statement is:

```sql
INSERT INTO customers (id, name, age) VALUES (1, 'John Doe', 30);

-- The script will generate the following UPDATE statement:
UPDATE customers SET name = 'John Doe', age = 30 WHERE id = 1;
```

### How to Use

- Insert your INSERT statements in the insert_statements variable in the script.
- Run the script.
- The generated UPDATE statements will be saved in transform-update.sql.

### Requirements

- Python 3.x
